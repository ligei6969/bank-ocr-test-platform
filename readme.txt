# 银行远程开户资料 OCR 识别与自动化测试平台

## 1. 项目定位

本项目是一个面向银行远程开户、信贷进件、信用卡申请等场景的 OCR 识别与自动化测试平台。

项目目标不是做一个复杂的“银行中台系统”，而是做一个结构清晰、功能闭环、适合写进简历和面试讲解的单体项目。它重点体现两类能力：

- 测试开发能力：接口测试、异常场景测试、回归测试、测试数据构造、测试报告输出。
- 图像算法能力：OCR 识别、图像质量检测、异常图片处理、字段级识别效果评估。

项目主线可以概括为：

```text
用户上传银行业务资料图片
    -> 图像质量检测
    -> OCR 文本识别
    -> 关键字段解析
    -> 银行业务规则校验
    -> 输出审核结果
    -> 自动化测试与评估
```

## 2. 业务场景

银行 App 在远程开户、信贷进件、信用卡申请等流程中，通常需要用户上传身份证、银行卡、开户申请表等图片资料。

系统需要自动判断图片是否清晰、是否反光、是否遮挡，并从图片中识别姓名、身份证号、银行卡号、手机号等关键字段。识别完成后，还要根据银行业务规则判断字段是否合法，最终输出通过、人工复核或拒绝等审核结果。

本项目模拟的就是这一类银行资料审核流程。

## 3. 核心功能

### 3.1 图像质量检测

检测用户上传图片是否适合 OCR 识别，重点覆盖：

- 模糊检测
- 亮度异常检测
- 反光检测
- 遮挡检测
- 旋转或倾斜检测

该模块用于在 OCR 前拦截低质量图片，降低识别错误率和人工审核成本。

### 3.2 OCR 识别

使用 PaddleOCR 作为 OCR 基线工具，对身份证、银行卡、申请表等图片进行文字识别。

项目重点不是从零训练 OCR 大模型，而是把 OCR 能力接入银行业务流程，完成接口封装、结果解析、异常输入处理和自动化测试评估。

### 3.3 字段解析

从 OCR 文本中提取结构化字段，例如：

- 姓名
- 身份证号
- 银行卡号
- 手机号
- 日期

该模块体现项目从“识别文字”到“业务可用数据”的转换。

### 3.4 规则校验

对解析后的字段做银行业务规则校验，例如：

- 身份证号长度、出生日期、校验位是否合法
- 银行卡号是否为纯数字，是否满足长度和 Luhn 校验
- 手机号格式是否合法
- OCR 结果与申请表字段是否一致
- 图片质量不合格时是否进入人工复核或拒绝流程

### 3.5 自动化测试

使用 Pytest 对核心模块和接口进行自动化测试，覆盖：

- 正常 OCR 识别
- 模糊图片
- 反光图片
- 遮挡图片
- 非法身份证号
- 非法银行卡号
- 空文件或损坏图片
- 审核结果分流逻辑

后续可扩展 Allure 测试报告和 Locust 性能测试。

## 4. 技术栈

第一版建议使用：

- Python
- FastAPI
- PaddleOCR
- OpenCV
- Pytest
- Requests

增强版可加入：

- Allure：生成自动化测试报告
- Locust：接口性能测试
- Docker：本地部署封装
- GitHub Actions：自动化回归测试

## 6. 第一阶段开发目标

第一阶段只做最小可用版本，不急着扩展复杂架构。

优先完成：

1. 生成身份证、银行卡、申请表的模拟图片数据。
2. 生成模糊、反光、遮挡、旋转等异常样本。
3. 实现图像质量检测函数。
4. 接入 PaddleOCR，返回 OCR 文本列表。
5. 从 OCR 文本中提取身份证号、银行卡号、手机号等字段。
6. 实现身份证号、银行卡号、手机号等规则校验。
7. 封装 `/ocr`、`/quality-check`、`/review` 三个接口。
8. 编写 Pytest 自动化测试用例，保证核心流程能跑通。

## 7. 推荐接口设计

```text
POST /ocr
```

上传图片，返回 OCR 识别文本。

```text
POST /quality-check
```

上传图片，返回图片质量检测结果。

```text
POST /review
```

上传图片，执行质量检测、OCR 识别、字段解析和规则校验，返回最终审核结果。

审核结果建议分为：

```text
pass    通过
review  人工复核
reject  拒绝
```

## 8. 测试重点

测试开发方向重点展示这些内容：

- 接口是否能正确处理正常图片。
- 模糊、反光、遮挡图片是否能被识别为低质量输入。
- OCR 结果为空或字段缺失时是否能正确返回错误。
- 身份证号、银行卡号、手机号等非法字段是否能被规则拦截。
- `/review` 接口是否能根据质量检测和规则校验输出正确审核结论。

建议第一版至少覆盖 15 条核心测试用例。

## 9. 简历表述方向

可以将项目写成：

> 基于 FastAPI、PaddleOCR、OpenCV、Pytest 构建银行远程开户资料 OCR 识别与自动化测试平台，支持身份证、银行卡、开户申请表等资料的 OCR 识别、图像质量检测、字段结构化与规则校验。构建合成测试数据集，并生成模糊、反光、遮挡、旋转等异常样本，用于验证 OCR 服务在异常输入下的稳定性。编写接口自动化测试用例，覆盖正常识别、低质量图片、字段缺失、非法证件号等场景，并输出测试结果。

## 10. 项目亮点

- 场景贴近银行远程开户和信贷进件业务。
- 不只是调用 OCR，而是完成了质量检测、字段解析、规则校验和测试评估闭环。
- 使用合成数据，避免真实用户隐私数据风险。
- 能同时体现图像算法能力和测试开发能力。
- 结构控制在单体项目范围内，适合快速做出可运行版本。
